## üì∑ Screenshots to Capture

### 1. Landing Page (Hero Section)
- **URL:** `http://localhost:3000/`
- **What to capture:** Full hero section with gradient background
- **Save as:** `screenshots/landing-page.png`
- **Tips:** 
  - Use full browser width (1920px recommended)
  - Capture the hero section with stats
  - Make sure animations have loaded

### 2. Map Workshop Interface
- **URL:** `http://localhost:3000/map-workshop`
- **What to capture:** Map with workshop markers and sidebar
- **Save as:** `screenshots/map-workshop.png`
- **Tips:**
  - Show multiple workshop markers on map
  - Include the sidebar with workshop list
  - Zoom to show a good area coverage

### 3. Mechanic Dashboard
- **URL:** `http://localhost:3000/mechanic-dashboard`
- **What to capture:** Dashboard with service requests
- **Save as:** `screenshots/mechanic-dashboard.png`
- **Tips:**
  - Login as a mechanic first
  - Show some sample requests
  - Capture the full dashboard layout

### 4. Customer Dashboard
- **URL:** `http://localhost:3000/dashboard`
- **What to capture:** Customer view with workshops
- **Save as:** `screenshots/customer-dashboard.png`
- **Tips:**
  - Login as a customer
  - Show workshop cards
  - Include search/filter options

### 5. AI Chatbot
- **URL:** `http://localhost:3000/` (wait 8 seconds for chatbot to appear)
- **What to capture:** Chatbot widget open with conversation
- **Save as:** `screenshots/ai-chatbot.png`
- **Tips:**
  - Wait for idle trigger or click chat icon
  - Have a sample conversation visible
  - Show the chatbot interface clearly

### 6. Service Request Form
- **URL:** `http://localhost:3000/request/[mechanicId]`
- **What to capture:** Service request form
- **Save as:** `screenshots/service-request.png`
- **Tips:**
  - Show the form partially filled
  - Highlight the urgency selector
  - Include location fields

### 7. Mechanic Profile/Settings
- **URL:** `http://localhost:3000/mechanic-profile`
- **What to capture:** Mechanic profile settings page
- **Save as:** `screenshots/mechanic-profile.png`
- **Tips:**
  - Show workshop details form
  - Include services offered section
  - Display profile image upload

## üìê Screenshot Best Practices

### Browser Setup
- **Resolution:** 1920x1080 (Full HD)
- **Browser:** Chrome or Edge (for consistency)
- **Zoom:** 100% (default)
- **Extensions:** Disable or use incognito mode

### Capture Tools

#### Windows:
- **Snipping Tool:** `Win + Shift + S`
- **Snip & Sketch:** Built-in Windows tool
- **ShareX:** Free, powerful screenshot tool

#### Browser Extensions:
- **Awesome Screenshot**
- **Fireshot**
- **Nimbus Screenshot**

### Quality Tips
1. ‚úÖ Use high resolution (at least 1920x1080)
2. ‚úÖ Capture full sections, not partial views
3. ‚úÖ Ensure text is readable
4. ‚úÖ Show real data, not "Lorem ipsum"
5. ‚úÖ Use consistent browser window size
6. ‚úÖ Disable browser extensions that add UI elements
7. ‚úÖ Clear browser console/dev tools
8. ‚úÖ Use light mode or dark mode consistently

## üí° Pro Tips

1. **Consistency:** Use the same browser and resolution for all screenshots
2. **Data:** Add some realistic sample data before capturing
3. **Timing:** Capture after animations have completed
4. **Cropping:** Crop to focus on the important parts
5. **Annotations:** Add arrows or highlights to draw attention (optional)
6. **File Size:** Compress images to under 500KB each
7. **Format:** Use PNG for UI screenshots (better quality than JPG)
